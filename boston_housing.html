Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

Question 7 - Answer¶

A machine learning model can have two types of parameters. The first type are parameters that are passed to the model for training. The second type are parameters for the "configuration" of the model itself. These parameters are also called hyper parameters.

The grid search technique helps us finding the optimal configuration of hyper parameters for a given model type. In sklearn, this is achieved by assigning a number of hyper parameter dictionaries to the variable parameter_grid like {'max_depth': [1,2,3]} and then creating a GridSearchCV object with parameter_grid as one of the constructor variables.

The object of GridSearchCV will then iterate over the combinations of various parameter dictionaries and the function best_params of GridSearchCV will output the set of parameters that fit best for the model.

Question 8 - Answer¶

One way of training a model is to split the data set into training and testing data. This ensures the golden rule that a model must never be trained on the test data. Given that we split out a data set into training and test data, it can happen that the accuracy of a number of consequtive tests varies (i.e. variance problem).

A mitigation to this problem is the k-fold cross-validation technique. This means that the data is divided into k-folds. The model is then trained on k-1 folds with the remaining fold being used as test data. The k-fold cross-validation technique iterates through each combination of folds and returns their accuracies at the end of the process.

By calculating the mean and the standard deviation of the accuracies it is possible to evaluate the performance of the model. A low standard deviation of the accuracies would mean low variance.

from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer
from sklearn.model_selection import GridSearchCV

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    cv_sets = ShuffleSplit(n_splits = 10, test_size = 0.20, random_state = 0)

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    md_range=range(1,10)
    params = dict(max_depth=md_range)

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Create the grid search cv object --> GridSearchCV()
    # Make sure to include the right parameters in the object:
    # (estimator, param_grid, scoring, cv) which have values 'regressor', 'params', 'scoring_fnc', and 'cv_sets' respectively.
    grid = GridSearchCV(regressor,params,scoring_fnc,cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_

Question 1: How relevant today is data that was collected from 1978? How important is inflation?

Data collected from 1978 is about 41 years old. In the section getting started is has been stated that the feature MEDV has been multiplicatively scaled to account for 35 years of market inflation. This scaling still misses the the development of prices for the last six years. Additionally, the features determining a home price might change over a period of 40 years. Question 2: Are the features present in the data sufficient to describe a home?

Three features for prediciting a target value like a home price are defintely a very small set. Other factors like low-barriers for elderly people, presence of pool, etc. should be added to the model.

Prices for homes in rural areas are determined by additional features like distance to public transport, size of the area the home is located on, etc. Therefore, data collected from urban cities are only applicable to a certain degree.
